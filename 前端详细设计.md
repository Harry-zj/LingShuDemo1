# 灵枢系统 Web 前端详细设计说明书

## 1. 项目概述

灵枢是一个高校综测全流程管理平台，前端基于 **Vue 3 + Vite 5** 构建，采用 Google Material You 米白柔和教务风格，覆盖学生材料提交、AI 智能填表、五维评定分析、个性化报告生成、信息管理与审核等完整业务闭环。

---

### 4.2 路由守卫

```js
router.beforeEach((to, from, next) => {
  const store = useUserStore()
  // 1. 未登录 → 重定向 /login
  // 2. 角色不在 meta.roles 中 → 重定向首页
  // 3. 否则放行
})
```

### 4.3 懒加载

所有页面组件通过 `() => import(...)` 动态导入，首次加载仅加载当前页面所需代码块。

## 6. 状态管理（Pinia userStore）

**存储字段**：
```js
{
  token: '',           // JWT Token
  user: {
    id: 0,
    username: '',
    role: '',          // student/admin/class_committee/counselor/student_affairs
    real_name: '',
    student_no: '',
    class_name: '',
    college: '',
    major: '',
    grade: ''
  }
}
```

**计算属性**：
- `isLoggedIn`：token 是否存在
- `userRole`：快捷获取当前角色

**方法**：
- `setAuth(token, user)`：登录时保存
- `logout()`：清除登录态
- 通过 `localStorage` 持久化，刷新不丢失

---

## 7. HTTP 请求层（Axios 封装）

**文件**：`client/src/api/request.js`

**功能**：
- `baseURL`: 空（通过 Vite proxy 转发 `/api` → `localhost:3000`）
- 请求拦截器：自动附加 `Authorization: Bearer <token>`
- 响应拦截器：统一解析 `{ code, msg, data }` 格式
- 错误处理：401 跳转登录页，其他 toast 提示

**Vite 代理配置**（`vite.config.js`）：
```js
server: {
  port: 5173,
  proxy: { '/api': { target: 'http://localhost:3000', changeOrigin: true } }
}
```

### 8.3 模块一：智能填表（zongce/）

#### 8.3.1 SmartFill.vue — AI 智能填表

**页面结构**：
- 顶部：页面标题 + 模板类型切换
- 三大子页签：智能匹配 / 批量填表 / 对话填表

**子组件 SmartFillForm.vue**：
- 上传区：点击或拖拽上传 .docx 模板，校验格式和文件大小
- 数据预览区（可折叠）：展示即将填入模板的数据（姓名/学号/F1/F2/F3 各分项）
- 操作区：一键填表 → 下载已填写文件
- 占位符指南（可折叠）：列出所有可用占位符及说明

**数据来源**：
- 从用户最新的 `assessment_forms.scores` 和 `assessment_form_items` 中读取
- 前端通过占位符映射数据到模板字段

#### 8.3.2 BatchFill.vue — 批量填表

- 上传 Excel 数据表 + Word 模板
- 配置字段映射关系
- 批量生成并打包下载

#### 8.3.2 SmartFill 子组件

**SmartFillRule.vue** — 规则管理面板
- 上传评分规则文件（docx/xlsx/pdf/png）或输入文本
- AI 解析为结构化规则项，按五维分组展示（计分规则/限制规则/冲突规则）
- 支持整组批量确认，SSE 实时显示解析进度

**SmartFillMaterial.vue** — 材料识别面板
- 创建材料条目，上传附件（图片/PDF/docx）
- 触发 AI 分析，展示识别结果（分类标签/建议分数/置信度）
- 逐一确认或忽略识别结果

**SmartFillScore.vue** — 评分清单面板
- 展示总分数 + 五维分项得分
- 按已确认材料逐行列出分类、分数和置信度

#### 8.3.3 ChatFill.vue — 对话式填表

- 上传模板 → AI 分析模板结构
- 对话界面逐字段交互填写
- 实时预览填充进度

| 历史发展 | HistoryTrend | 5个统计卡片（始终可见）+ 详细图表（折叠） |

**右 38%**：

| 模块 | 说明 |
|------|------|

#### 8.4.2 ReportView.vue — 个性化评定报告

**定位**：定性诊断 + 成长规划 + 个人档案（与数据总览内容不重复）

**布局**：65/35 双栏，右侧 `position: sticky`

**左 65%**：

| 序号 | 模块 | 功能 |
|------|------|------|
| 1 | 学生档案卡片 | 姓名/年级/专业/辅导员/测评学年 + 等级/优势/薄弱标签 + 发展特征标签 |
| 2 | AI 综合总评 | DeepSeek 生成 300-500 字三段式评语（总览→逐维分析→总结），右上角重新生成按钮 |
| 3 | 分阶段提升方案 | 左48%「本月/期末前」+ 虚线分隔 + 右48%「学年内」，空心勾选框可点击标记完成 |
| 4 | 成长目标 | 左30% 五维预览（点击切换） + 右70% 记事本编辑器，按批次独立存储 |
| 5 | 历年成长回顾 | 文字时间轴，按选中批次学年过滤（选大二只显示大一+大二） |

**右 35%**：

| 序号 | 模块 | 功能 |
|------|------|------|
| 1 | 五维个性画像 | 5 张维度卡片，含分数+等级+优/弱标签，本地模板生成（根据实际分项得分不同文字不同），点击跳转维度活动指南 |
| 2 | 本学年亮点记录 | 基于真实加分事件生成 |
| 3 | 待提升事项 | B1-B8 低分项识别和提醒 |
| 4 | 操作功能区 | 导出完整PDF / 打印 / 保存规划 / 分享报告 / 生成年度总结 |

**数据加载策略**：
- 基础数据（评价/历史）→ 并行加载
- AI 内容 → 首次进入自动生成 → 服务端缓存 → 非首次秒开
- 维度画像 → 本地模板即时生成（不调 AI，根据分数动态拼文）
- 切换批次 → 独立缓存，互不影响

#### 8.4.3 DimensionDetail.vue — 维度活动指南

**布局**：左侧 50% 图片轮播区 + 右侧活动卡片列表

**图片轮播区**：
- 有图片时：自动播放（3.5s/张）+ 左右箭头手动切换 + 底部圆点指示器
- 无图片时：虚线占位框 + 提示文字
- 图片路径：`/images/{维度}/{图片名}`

**右侧活动卡片**：
- 每张卡片：序号圆圈 + 标题 + 分类标签（如"B1 证书"）+ 描述 + 官网链接（可点击）+ 添加至目标按钮

**活动选取算法**：
- 基于 `用户ID + 维度` 作为随机种子 → 同一用户永远看到相同的活动
- 有图片的活动优先入选
- 选取结果缓存到 localStorage，版本号控制更新

### 8.5 模块三：信息管理（module3/）

#### 8.5.1 StudentDashboard.vue — 学生端（我的综测）

**功能**：
- 批次列表 + 倒计时
- 综测评价表展示（assessment_forms）
- 支撑材料按 F1(A1-A5) / F2(课程成绩) / F3(B1-B8) 分类
- 修改项目名称、计分理由、分值
- 提交至班级测评小组审核

**状态流转（前端展示）**：
```
smart_ready → pending_class_committee → pending_counselor
    → pending_student_affairs → approved / rejected
```

#### 8.5.2 ClassLeaderDesk.vue — 待审核列表

**功能**：
- 按当前角色显示待评价学生列表（`getPendingReviews`）
- 学生卡片：姓名/学号/班级/分数/等级/当前状态
- 点击进入审核详情

**角色-状态映射**：
- 班级测评小组 → `pending_class_committee`
- 辅导员 → `pending_counselor`
- 学生工作处 → `pending_student_affairs`

#### 8.5.3 ReviewDetail.vue — 材料审核详情

**功能**：
- 学生综测表全貌（`getFormDetail`）
- 顶部分数+等级，中部 F1/F2/F3 分类明细
- 审核操作：通过 / 退回修改 / 不予认定
- 等级手动调整（班委和辅导员可通过时设置等级）
- 审核记录时间线（`assessment_review_records`）

**审核流转逻辑**：
```
class_committee 通过 → pending_counselor
counselor 通过 → pending_student_affairs
student_affairs 通过 → approved（最终认定）
任意角色退回 → returned_by_xxx（学生修改后重新提交）
任意角色拒绝 → rejected（不予认定）
```

#### 8.5.4 BatchManage.vue — 批次管理（管理员端）

**功能**：
- 创建/编辑/删除测评批次
- 设置批次时间（开始/截止）
- 配置是否允许学生编辑、退回重交、评审意见要求
- 批次状态管理：草稿→发布→关闭→归档

#### 8.5.5 TeacherConsole.vue — 统计总览（管理员端）

**功能**：
- 各状态学生数量统计
- 各等级分布
- 导出 CSV 汇总表

#### 8.5.6 AssessmentFormPanel.vue — 综测评价表面板（共享组件）

嵌入 StudentDashboard、ReviewDetail、ReviewerDesk 中复用。
- F1/F2/F3 三分类评分表，含 A1-A5、B1-B8 子项明细

## 11. 五维评分计算（scoreHelper.js）

**唯一公式来源（Single Source of Truth）**，前后端共享逻辑。

**原始加权和（未归一化）**：
```js
德 = A1×0.25 + A2×0.25 + A3×0.25 + A4×0.25    (满分20)
智 = F2×0.4 + B1×0.2 + B2×0.2 + B3×0.2        (满分58)
体 = A5×0.4 + B7×0.6                            (满分26)
美 = B4×1.0                                      (满分30)
劳 = B6×0.5 + B8×0.3 + B5×0.2                   (满分30)
```

**归一化**：各维度 `= clamp(原始加权 / 该维度满分 × 100, 0, 100)`

**等级判定**：
```js
≥90: 优秀(绿)  ≥80: 良好(蓝)  ≥70: 中等(橙)  ≥60: 合格(红)  <60: 待提升(灰)
```

| 数据类型 | 存储位置 | 键格式 | 跨设备 | 更新时机 |
|---------|---------|--------|--------|---------|

## 13. 关键交互流程

### 13.1 综测数据总览 → 切换批次
```
用户选择批次下拉 → loadData()
  → getBatches() 获取批次列表
  → getEvaluation({ batch_id }) 获取该批次评定数据
  → result 更新 → 雷达图/柱状图/排名全部联动刷新
  → HistoryTrend 按 activeBatchTitle 过滤历史数据
```

### 13.2 个性报告 → AI 生成
```
页面加载 → loadData()
  → loadCache(batch_id) 查服务端缓存
  → 有缓存 → reportData = 缓存内容
  → 无缓存 → handleGenerate() → generateReport API → DeepSeek 生成 → 保存到服务端
  → 所有 computed (shortTermPlan/longTermPlan/highlights) 自动更新
```

### 13.3 维度指南 → 添加至成长目标
```
点击「添加至目标」 → localStorage 桥接存储
  → 打开个性报告页面 → loadGoals() 检测桥接数据
  → 合并到 _cache.goals → saveCache() 写入服务端
  → 成长目标编辑器自动展示
```

### 13.4 审核流程（模块三）
```
学生提交 → status = pending_class_committee
班级测评小组审核 → 通过 → pending_counselor / 退回 → returned_by_xxx
辅导员审核 → 通过 → pending_student_affairs / 退回
学生工作处审核 → 通过 → approved（最终认定）/ 拒绝 → rejected
```

## 16. 性能优化

| 策略 | 说明 |
|------|------|
| 路由懒加载 | 所有页面 `() => import(...)` |
| 图表按需初始化 | ECharts 仅在容器可见时 init，折叠状态跳过 |

| 服务端缓存 | AI 生成内容存入 MySQL，跨设备共享，避免重复生成 |
| localStorage 缓存 | 维度活动选择缓存，秒开 |
| 并行加载 | `Promise.all` 并行获取 evaluation + advice + history |

## 17. 错误处理

| 场景 | 处理方式 |
|------|---------|
| API 401 | Axios 拦截器自动跳转登录页 |
| API 403 | 提示"权限不足" |
| API 其他错误 | toast 提示错误信息 |

| 图片加载失败 | 回退到虚线占位框 |
