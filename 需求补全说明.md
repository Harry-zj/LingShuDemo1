# 需求补全说明

本次补全以新版《综测管理平台需求分析报告》为准，聚焦单一综测场景：学生材料提交、班干部初审、老师复核、统计导出、通知日志和权限控制。

## 当前运行模式

- 已临时禁用真实数据库连接。
- 后端使用 `server/src/mock/assessmentStore.js` 作为内存数据仓库。
- 服务重启后数据会恢复为演示数据。
- 演示账号如下，密码均为 `123456`：
  - `student`：学生
  - `leader`：班干部
  - `teacher`：老师

## 已覆盖的需求点

1. 用户与班级管理
   - 支持学生、班干部、老师登录。
   - 老师可查看班级、学生和班干部信息。
   - 后端补充了设置班干部接口。

2. 综测批次管理
   - 老师可创建、编辑、发布、关闭、归档综测批次。
   - 批次包含名称、说明、开始时间、截止时间、材料要求和状态。

3. 材料提交
   - 学生可保存草稿、上传附件、正式提交。
   - 学生可修改草稿、班干部退回、老师退回状态下的材料。
   - 正式提交后进入“待班干部审核”。

4. 材料审核
   - 班干部查看本班待初审材料。
   - 班干部可通过、退回、驳回。
   - 老师查看待复核材料。
   - 老师可通过、退回、驳回。
   - 每次审核会记录审核意见。

5. 附件管理
   - 支持图片、PDF、Word、Excel。
   - 使用上传中间件限制文件大小和文件类型。
   - 附件会关联到材料，列表中可查看附件数量与名称。

6. 统计导出
   - 支持统计学生总数、已提交人数、未提交人数、待审核数、通过数、退回数、驳回数。
   - 支持查看班级进度和未提交名单。
   - 当前以 CSV 形式导出，可直接用 Excel 打开。

7. 通知与日志
   - 审核通过、退回、驳回时生成学生通知。
   - 学生提交后通知班干部。
   - 班干部初审通过后通知老师。
   - 关键操作会写入内存日志，老师端可查看。

## 后续接入数据库建议

当前内存数据仓库已经按核心实体组织：用户、班级、批次、材料、附件、审核记录、通知、操作日志。后续可将 `assessmentStore.js` 中的方法迁移到 service/DAO，并恢复 `server/src/config/database.js` 的 MySQL 连接。
