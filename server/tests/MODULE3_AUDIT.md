# 模块三提交与评测系统审查及关键修复说明

## 本次修复范围

本次只修复以下三个已确认问题：

1. 复核分值未写回最终结果。
2. 单独等级修改接口存在越权路径。
3. 智能填表来源未按综测批次隔离。

未修改此前审查中发现的其他问题，也未调整无关页面样式、其他模块业务逻辑或接口数据格式。

## 已修复内容

### 1. 复核分值生效

- 评价操作为“通过”时，将逐项复核分值写回 `assessment_form_items.score`。
- 写回后重新汇总 F1、F2、F3 和加权总分。
- 同步更新 `assessment_forms.scores` 与自动等级 `assessment_forms.level`。
- 异议复评使用同一逻辑，因此复评调整会进入最终结果。
- 复核分值限制为 0 至 999.99 的有限数字。
- “退回”和“不予认定”不会提前覆盖学生当前项目分值。

### 2. 等级接口权限收紧

- `PUT /forms/:id/level` 不再允许学生调用。
- 辅导员只能在表单处于 `pending_counselor`、学生属于本人管辖范围、且本人持有该表单待评任务时调整等级。
- 学生工作处只能在表单处于 `pending_student_affairs`、且本人持有该表单待评任务时调整等级。
- 提交的等级必须存在于当前系统等级规则中；空值表示恢复自动等级。

### 3. 批次数据严格隔离

- `fill_results` 新增 `batch_id` 和联合索引。
- 生成智能填表 Word 结果时记录所选批次。
- 模块三只读取当前学生、当前批次的填表结果。
- 只读取当前批次已发布的规则集。
- 填表预览与材料匹配结果按当前规则集过滤。
- Word 结果元数据同时校验学生和批次。
- 历史 `batch_id` 为空的填表结果不会自动猜测归属；需要在对应批次重新生成一次填表结果后再导入，避免串用旧批次数据。
